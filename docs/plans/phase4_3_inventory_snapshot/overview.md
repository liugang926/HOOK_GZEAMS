# Phase 4.3: 盘点快照和差异处理 - 总览

## 概述

实现盘点快照机制，确保盘点期间资产数据不受业务变更影响，自动识别盘点差异，支持差异处理流程。

---

## 1. 业务背景

### 1.1 快照必要性

| 问题 | 说明 |
|------|------|
| **数据不一致** | 盘点期间资产被调拨，导致数据不一致 |
| **差异无法追溯** | 无快照无法确认盘点时资产状态 |
| **重复盘点** | 两次盘点结果无法独立对比 |

### 1.2 核心设计

- **不可变快照**：快照一旦生成就不可修改
- **差异自动识别**：自动对比快照与实际盘点
- **差异处理流程**：差异需要审批后才能调整

## 3. 数据模型

### 3.1 InventorySnapshot（盘点快照）

| 字段 | 说明 |
|------|------|
| task | 关联盘点任务 |
| asset | 资产 |
| snapshot_data | 快照数据(JSON) |
| snapshot_at | 快照时间 |
| is_immutable | 不可变标记 |

### 3.2 快照数据结构

```json
{
    "asset_code": "ZC20240115001",
    "asset_name": "MacBook Pro",
    "location": "A区3号柜",
    "custodian": "张三",
    "department": "技术部",
    "asset_status": "in_use",
    "snapshot_at": "2024-01-15T10:00:00Z"
}
```

---

## 4. 差异类型

### 4.1 差异分类

| 差异类型 | 代码 | 说明 |
|----------|------|------|
| 盘亏 | loss | 快照有但未盘点 |
| 盘盈 | surplus | 盘点有但快照无 |
| 位置不符 | location_mismatch | 位置不一致 |
| 保管人不符 | custodian_mismatch | 保管人不一致 |
| 状态不符 | status_mismatch | 状态不一致 |

### 4.2 差异严重程度

| 级别 | 说明 | 处理要求 |
|------|------|----------|
| 严重 | 盘亏/盘盈 | 必须审批 |
| 一般 | 位置/保管人不符 | 确认即可 |
| 轻微 | 状态不符 | 记录即可 |

## 5. 差异处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    差异处理流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  1. 盘点完成提交                                             │
│     ↓                                                         │
│  2. 系统自动生成差异清单                                     │
│     ↓                                                         │
│  3. 盘点人确认差异                                           │
│     ↓                                                         │
│  4. 提交差异处理申请                                         │
│     ↓                                                         │
│  5. 审批人审批                                               │
│     ├── 批准 → 执行调整                                      │
│     └── 拒绝 → 驳回修改                                      │
│     ↓                                                         │
│  6. 执行资产调整                                             │
│     ↓                                                         │
│  7. 生成盘点报告                                             │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

## 6. API接口

### 6.1 生成快照

```
POST /api/inventory/tasks/{id}/snapshot/
Response: {
    "snapshot_id": 100,
    "asset_count": 500,
    "created_at": "2024-01-15T10:00:00Z"
}
```

### 6.2 检测差异

```
POST /api/inventory/tasks/{id}/detect-differences/
Response: {
    "differences": [
        {
            "type": "missing",
            "asset_code": "ZC001",
            "severity": "high"
        }
    ]
}
```

### 6.3 处理差异

```
POST /api/inventory/differences/{id}/resolve/
Request: {
    "action": "approve",
    "resolution": "write_off",
    "comment": "同意报损"
}
```
