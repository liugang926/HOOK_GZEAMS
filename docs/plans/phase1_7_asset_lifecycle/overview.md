# Phase 1.7: 资产生命周期管理 - 总览

## 1. 功能概述与业务场景

### 1.1 功能概述

本模块实现固定资产从需求提出到报废处置的完整生命周期管理，确保资产在各个业务环节的状态可追踪、价值可计算、责任可追溯。涵盖以下核心业务流程：

1. **采购申请管理**：员工发起资产需求申请，经过审批流程后推送至M18采购系统，实现从需求到采购的全流程管理
2. **资产入库验收**：资产到货后进行质量检验和数量核对，验收通过后自动生成资产卡片，确保账实相符
3. **维护保养管理**：支持故障报修、维修派单、保养计划制定与任务执行，延长资产使用寿命
4. **报废处置管理**：支持多种处置方式（报废/出售/捐赠/调拨/销毁），包含技术鉴定、审批执行和财务凭证生成

### 1.2 业务场景

| 场景 | 用户角色 | 业务流程 | 预期效果 |
|------|---------|---------|---------|
| **资产采购** | 普通员工→部门主管→资产管理员 | 发起需求申请→部门审批→资产审批→推送M18采购系统 | 采购流程合规化，需求可追溯 |
| **资产入库** | 资产管理员→质检员 | 创建验收单→质检确认→数量核对→生成资产卡片 | 确保入库资产质量合格，信息完整 |
| **故障维修** | 普通员工→资产管理员→维修人员 | 故障报修→派单→维修执行→验收完成 | 资产快速恢复正常使用状态 |
| **定期保养** | 资产管理员→指定执行人 | 制定保养计划→定时生成任务→执行保养→记录结果 | 延长资产使用寿命，降低故障率 |
| **资产处置** | 资产管理员→财务→管理层 | 发起报废申请→技术鉴定→审批→执行处置→生成凭证 | 规范处置流程，实现资产残值回收 |

## 2. 用户角色与权限

| 角色 | 权限说明 | 主要操作 |
|------|---------|---------|
| **普通员工** | 发起采购申请、故障报修、查看自己的申请记录 | 创建采购申请、创建维修申请、查看我的申请 |
| **部门主管** | 审批本部门的采购申请和报废申请 | 审批采购申请、审批报废申请、查看部门统计 |
| **资产管理员** | 资产验收、维护派单、保养计划管理、报废鉴定 | 创建验收单、维护派单、制定保养计划、报废鉴定 |
| **维修人员** | 接收维修工单、执行维修任务、完成任务验收 | 查看派单、执行维修、填写维修记录、完成任务 |
| **财务人员** | 查看采购预算、确认报废处置、生成财务凭证 | 查看采购预算、审批处置方案、生成处置凭证 |
| **系统管理员** | 全部权限，包括配置管理、数据维护 | 配置管理、数据维护、系统监控 |

## 8. 业务背景

### 1.1 当前痛点

| 痛点 | 说明 | 影响 |
|------|------|------|
| **采购缺失** | 资产从需求到采购过程无系统记录 | 采购合规性无法保障 |
| **入库缺失** | 资产验收后直接投入使用，无正式入库流程 | 资产信息不完整，追溯困难 |
| **维护缺失** | 资产维修保养无记录 | 维护成本无法统计，资产状态不明 |
| **处置缺失** | 报废处置流程不规范 | 资产残值损失，合规风险 |

### 1.2 业务目标

- **全流程可追溯**：资产从需求到处置的每个环节都有系统记录
- **状态自动流转**：资产状态随业务流程自动更新
- **价值准确计量**：实时计算资产净值、折旧、残值
- **审批合规可控**：关键环节支持审批流，确保合规性

## 3. 功能范围

### 3.1 采购申请 (Purchase Request)

| 功能点 | 说明 |
|--------|------|
| 需求单创建 | 员工发起资产需求申请 |
| 需求审批 | 多级审批流程（部门主管 → 资产管理员 → 财务） |
| 采购对接 | 审批通过后推送至M18采购系统 |
| 状态同步 | 采购进度实时同步 |

### 3.2 资产入库 (Asset Receipt)

| 功能点 | 说明 |
|--------|------|
| 验收单创建 | 资产到货后创建验收单 |
| 验收检查 | 质检、数量核对、规格确认 |
| 资产卡片生成 | 验收通过后自动生成资产卡片 |
| 入库确认 | 正式登记入账 |

### 3.3 维护保养 (Maintenance)

| 功能点 | 说明 |
|--------|------|
| 维修记录 | 故障报修、维修过程、维修费用 |
| 保养计划 | 按周期生成保养任务 |
| 保养执行 | 保养记录、费用统计 |
| 状态更新 | 维修后更新资产状态 |

### 3.4 报废处置 (Disposal)

| 功能点 | 说明 |
|--------|------|
| 报废申请 | 发起资产报废申请 |
| 技术鉴定 | 评估资产残值、使用年限 |
| 审批流程 | 多级审批（资产管理 → 财务 → 管理层） |
| 处置执行 | 出售、捐赠、销毁等 |
| 凭证生成 | 自动生成处置财务凭证 |
