## 1. 业务背景与痛点

### 1.1 业务场景

在研发型、项目型企业中，大量资产是按项目进行分配和使用的：

| 场景 | 说明 |
|------|------|
| **项目立项** | 新项目启动时，需要分配相应的资产（设备、软件许可等） |
| **资产调配** | 从现有资产中调配给项目使用 |
| **新增采购** | 项目专用资产申请新增采购 |
| **项目执行** | 项目执行过程中资产的使用、维护、转移 |
| **项目结项** | 项目结束后资产回收、处置或转移 |

### 1.2 现状痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **资产归属不清** | 不知道哪些资产分配给了哪个项目 | 资产利用率低，重复采购 |
| **成本核算困难** | 无法统计项目的资产成本 | 项目盈利分析不准确 |
| **资产回收失控** | 项目结束后资产未及时回收 | 资产闲置浪费 |
| **跨项目调配混乱** | 资产在项目间转移无记录 | 权责不清 |

### 1.3 解决方案

建立资产与项目的关联关系，实现：

1. **项目资产分配** - 将资产分配给项目，记录分配日期、责任人
2. **项目资产视图** - 按项目查看资产清单和使用情况
3. **项目成本核算** - 统计项目占用的资产价值（折旧分摊）
4. **项目结束回收** - 项目结项时资产回收流程
5. **跨项目转移** - 资产在项目间转移的记录和审批

## 2. 用户角色与权限

| 角色 | 权限范围 | 说明 |
|------|---------|------|
| **项目经理** | 创建项目、分配/收回资产、查看项目资产、发起项目结项 | 核心角色 |
| **资产管理员** | 审批资产分配、查看所有项目资产、跨项目调配 | 审批角色 |
| **部门负责人** | 审批项目立项、审批项目结项 | 审批角色 |
| **财务人员** | 查看项目资产成本、折旧分摊数据 | 查询角色 |
| **项目成员** | 查看所在项目资产 | 查询角色 |

## 6. 核心概念

### 6.1 项目状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| planning | 筹备中 | 编辑项目信息、添加成员 |
| active | 进行中 | 分配资产、添加成员、资产转移 |
| suspended | 暂停 | 不可分配新资产 |
| completed | 已完成 | 查看项目信息、资产结算 |
| cancelled | 已取消 | 仅查看 |

### 6.2 资产分配类型

| 类型 | 说明 | 资产状态 |
|------|------|---------|
| permanent | 永久分配（专用） | 项目专用，不可他用 |
| temporary | 临时分配（共享） | 可共享，用完可归还 |
| shared | 共享分配 | 多项目共享 |

### 6.3 资产归还状态

| 状态 | 说明 |
|------|------|
| in_use | 使用中（未归还） |
| returned | 已归还到库存 |
| transferred | 已转移到其他项目 |

## 7. 非功能性需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 列表查询 | < 2s |
| 项目资产加载 | < 3s |
| 成本计算 | < 5s |

### 7.2 数据安全

| 要求 | 说明 |
|------|------|
| 数据隔离 | 项目数据受组织隔离控制 |
| 操作审计 | 资产分配/回收必须有完整审计 |
| 权限控制 | 项目成员只能查看本项目的资产 |
