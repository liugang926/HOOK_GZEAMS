# 06. 通知消息模块 - 详细设计

## 概述

通知消息模块负责向用户推送各类系统通知（审批通知、任务分配、提醒事项等），支持消息分类聚合、快捷操作、已读管理等。

---

## 1. 消息分类与聚合

### 1.1 消息类型

| 类型编码 | 类型名称 | 图标 | 优先级 |
|---------|---------|------|--------|
| approval | 审批通知 | 📋 | 高 |
| task | 任务分配 | 📝 | 高 |
| reminder | 提醒事项 | ⏰ | 中 |
| system | 系统通知 | 🔔 | 低 |
| announcement | 公告通知 | 📢 | 中 |

## 2. 消息快捷操作

### 2.1 消息卡片操作

```
┌─────────────────────────────────────────────────────────────────┐
│  📋 审批通知                                                     │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 张三的领用申请需要您的审批                                   ││
│  │  • 单号: LY202401001                                        ││
│  │  • 资产: MacBook Pro x1, Dell显示器 x1                       ││
│  │  • 原因: 新员工入职办公设备配置                               ││

│  │ 10分钟前                                            [查看详情]││
│  │                                                             ││
│  │ [同意] [拒绝] [稍后处理]                                    ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 快捷操作按钮

| 消息类型 | 快捷操作 | 说明 |
|---------|---------|------|
| 审批通知 | 同意、拒绝、稍后处理 | 直接操作无需跳转 |
| 任务分配 | 开始任务、延期 | 直接开始或延期任务 |
| 到期提醒 | 确认收到、查看详情 | 确认已收到提醒 |
| 系统通知 | 确定、查看详情 | 基础确认操作 |

## 5. API设计

### 5.1 获取消息列表

```http
GET /api/notifications/

Query Parameters:
  - type: string             # 消息类型筛选
  - status: string           # 状态筛选 (all | unread | read)
  - page: integer
  - page_size: integer

Response:
{
  "summary": {
    "total_count": 56,
    "unread_count": 12,
    "by_type": {
      "approval": 5,
      "task": 8,
      "reminder": 2,
      "system": 41
    }
  },
  "groups": [
    {
      "group_name": "今天",
      "group_time": "2024-01-10",
      "unread_count": 2,
      "items": [
        {
          "id": "notif_001",
          "type": "approval",
          "type_label": "审批通知",
          "title": "张三的领用申请需要您的审批",
          "content": {
            "request_no": "LY202401001",
            "applicant": "张三",
            "items": "MacBook Pro x1, Dell显示器 x1"
          },
          "is_read": false,
          "is_important": false,
          "created_at": "2024-01-10T10:20:00Z",
          "available_actions": ["approve", "reject", "view_detail"]
        }
      ]
    }
  ]
}
```

### 5.2 消息操作

```http
# 标记已读
POST /api/notifications/{id}/read/

# 批量标记已读
POST /api/notifications/batch-read/

# 快捷操作
POST /api/notifications/{id}/quick-action/

Request:
{
  "action": "approve"  # approve | reject | accept | dismiss
}

# 删除消息
DELETE /api/notifications/{id}/
```
