# Phase 2.4: 组织架构增强与数据权限 - 总览

## 概述

增强组织架构模块，支持一人多部门、部门负责人、完整部门路径显示等功能，并基于组织架构实现数据权限控制。

---

## 1. 业务背景

### 1.1 当前痛点

| 痛点 | 影响 |
|------|------|
| 一人只能属于一个部门 | 无法处理兼职/借调场景 |
| 无部门负责人概念 | 无法按部门分配审批 |
| 部门路径不完整 | 无法展示完整层级 |
| 无数据权限控制 | 用户可见范围过大 |

### 1.2 核心目标

- 支持一人多部门
- 支持设置部门负责人
- 显示完整部门路径
- 基于部门的数据权限

## 3. 数据模型增强

### 3.1 Department（部门）扩展

| 新增字段 | 说明 |
|----------|------|
| leader | 部门负责人（FK） |
| full_path | 完整路径（如：总部/技术部/后端组） |
| level | 层级（从1开始） |
| sort_order | 排序号 |

### 3.2 UserDepartment（用户部门关系）

| 字段 | 说明 |
|------|------|
| user | 用户 |
| department | 部门 |
| is_primary | 是否主部门 |
| is_leader | 是否负责人 |
| job_title | 职务 |

## 4. 数据权限设计

### 4.1 权限范围

| 范围 | 说明 | 示例 |
|------|------|------|
| 全部数据 | 管理员 | 所有资产 |
| 本部门及下级 | 部门负责人 | 技术部及其子部门资产 |
| 本部门 | 普通用户 | 本部门资产 |
| 本人 | 仅本人 | 自己创建的数据 |

### 4.2 权限计算

```python
def get_user_data_scope(user, resource_type):
    """获取用户数据范围"""

    # 1. 检查是否管理员
    if user.is_superuser:
        return 'all'

    # 2. 检查是否部门负责人
    leader_depts = get_led_departments(user)
    if leader_depts:
        return get_dept_with_children(leader_depts)

    # 3. 普通用户 - 本部门
    primary_dept = get_primary_department(user)
    return primary_dept.id
```

## 6. API接口

### 6.1 部门树（含完整路径）

```
GET /api/organizations/departments/tree/
Response: {
    "id": 1,
    "name": "技术部",
    "full_path": "总部/技术部",
    "level": 2,
    "leader": {...},
    "children": [...]
}
```

### 6.2 用户部门列表

```
GET /api/organizations/users/{id}/departments/
Response: {
    "results": [
        {
            "department": {...},
            "is_primary": true,
            "is_leader": false,
            "job_title": "工程师"
        }
    ]
}
```

### 6.3 设置主部门

```
POST /api/organizations/users/{id}/set-primary-department/
Request: {
    "department_id": 5
}
```
