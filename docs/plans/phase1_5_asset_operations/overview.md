# Phase 1.5: 资产领用/调拨/退库业务 - 总览

## 概述

实现资产领用、调拨、借用、退库等日常运营业务单据管理，记录资产在组织内部的流动轨迹。

---

## 1. 业务背景

### 1.1 业务场景

| 业务 | 场景 | 流程 |
|------|------|------|
| **资产领用** | 员工领用资产用于工作 | 申请→审批→发放→登记 |
| **资产借用** | 临时借用资产 | 申请→审批→借出→归还 |
| **资产调拨** | 部门间资产转移 | 申请→审批→调拨→确认 |
| **资产退库** | 离职或项目结束退回 | 申请→审批→退库→入库 |

### 1.2 核心目标

- 完整记录资产流动轨迹
- 支持审批流程控制
- 自动更新资产保管信息
- 支持批量操作

## 3. 单据类型

### 3.1 资产领用单（AssetPickup）

| 字段 | 说明 |
|------|------|
| pickup_no | 领用单号（L+YYYYMMDD+流水号） |
| applicant | 申请人 |
| department | 领用部门 |
| pickup_date | 领用日期 |
| pickup_reason | 领用原因 |
| items | 领用明细（资产列表） |
| status | 单据状态 |

### 3.2 资产调拨单（AssetTransfer）

| 字段 | 说明 |
|------|------|
| transfer_no | 调拨单号（D+YYYYMMDD+流水号） |
| from_dept | 调出部门 |
| to_dept | 调入部门 |
| transfer_date | 调拨日期 |
| transfer_reason | 调拨原因 |
| items | 调拨明细 |
| status | 单据状态 |

### 3.3 资产借用单（AssetLoan）

| 字段 | 说明 |
|------|------|
| loan_no | 借用单号（J+YYYYMMDD+流水号） |
| borrower | 借用人 |
| loan_date | 借用日期 |
| return_plan_date | 计划归还日期 |
| loan_reason | 借用原因 |
| items | 借用明细 |
| status | 单据状态 |

### 3.4 资产退库单（AssetReturn）

| 字段 | 说明 |
|------|------|
| return_no | 退库单号（R+YYYYMMDD+流水号） |
| returner | 退库人 |
| return_date | 退库日期 |
| return_reason | 退库原因 |
| items | 退库明细 |
| status | 单据状态 |

## 4. 单据状态机

### 领用单状态

```
draft（草稿）
   ↓
pending（待审批）
   ↓
approved（已批准）
   ↓
completed（已完成）
   ↓
returned（已退库）←可选
```

### 调拨单状态

```
draft（草稿）
   ↓
pending（待审批）
   ↓
approved（已批准）
   ↓
transferring（转移中）
   ↓
completed（已完成）
```

### 借用单状态

```
draft（草稿）
   ↓
pending（待审批）
   ↓
approved（已批准）
   ↓
loaned（已借出）
   ↓
returned（已归还）
```

---

## 5. 业务规则

### 5.1 领用规则

- 在库资产才能领用
- 调拨中的资产不能领用
- 借用中的资产不能领用
- 领用后自动更新保管人

### 5.2 调拨规则

- 需要双方部门负责人审批
- 调拨后更新使用部门
- 调拨后更新存放位置
- 支持批量调拨

### 5.3 借用规则

- 限制借用时长（如30天）
- 超期自动提醒
- 归还时检查资产状态
- 损坏需赔偿
