# Phase 1.2: Organizations 独立模块 - 总览

## 概述

将组织和部门相关功能从 accounts 模块独立出来，创建专门的 organizations 模块，支持多组织架构、部门树形结构、一人多部门等核心功能。

---

## 1. 业务背景

### 1.1 模块拆分原因

| 原因 | 说明 |
|------|------|
| **职责分离** | 组织架构与用户认证是两个独立领域 |
| **多组织支持** | SaaS模式需要多租户组织隔离 |
| **复杂关系** | 一人多部门、部门层级等复杂关系 |
| **扩展性** | 便于后续扩展（组织权限、数据权限） |

### 1.2 核心功能

- 组织/公司管理
- 部门树形结构
- 用户部门关联
- 部门负责人设置

## 3. 数据模型

### 3.1 Organization（组织）

| 字段 | 说明 |
|------|------|
| code | 组织编码 |
| name | 组织名称 |
| name_en | 英文名称 |
| parent | 上级组织（支持集团架构） |
| level | 层级 |
| sort_order | 排序 |
| is_active | 是否启用 |

### 3.2 Department（部门）

| 字段 | 说明 |
|------|------|
| organization | 所属组织 |
| code | 部门编码 |
| name | 部门名称 |
| parent | 上级部门 |
| full_path | 完整路径 |
| level | 层级 |
| leader | 部门负责人 |
| sort_order | 排序 |

### 3.3 UserDepartment（用户部门）

| 字段 | 说明 |
|------|------|
| user | 用户 |
| department | 部门 |
| is_primary | 是否主部门 |
| is_leader | 是否负责人 |
| job_title | 职务 |

## 4. 组织层级

### 4.1 集团架构

```
集团 (一级)
  ├── 子公司A (二级)
  │   ├── 部门A1 (三级)
  │   └── 部门A2 (三级)
  └── 子公司B (二级)
      ├── 部门B1 (三级)
      └── 部门B2 (三级)
```

### 4.2 部门树

```
总部 (一级)
  ├── 技术部 (二级)
  │   ├── 后端组 (三级)
  │   └── 前端组 (三级)
  ├── 财务部 (二级)
  └── 人事部 (二级)
```

## 5. API接口

### 5.1 组织列表

```
GET /api/organizations/
Response: {
    "results": [
        {
            "id": 1,
            "code": "HQ",
            "name": "集团总部",
            "level": 1
        }
    ]
}
```

### 5.2 部门树

```
GET /api/organizations/departments/tree/
Response: {
    "id": 1,
    "name": "总部",
    "children": [
        {
            "id": 2,
            "name": "技术部",
            "children": [...]
        }
    ]
}
```

### 5.3 用户部门列表

```
GET /api/organizations/users/{id}/departments/
Response: {
    "results": [
        {
            "department": {...},
            "is_primary": true,
            "job_title": "工程师"
        }
    ]
}
```
