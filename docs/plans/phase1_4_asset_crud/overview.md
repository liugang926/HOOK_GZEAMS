# Phase 1.4: 资产卡片CRUD - 总览

## 概述

实现资产卡片的完整CRUD功能，是固定资产管理的核心模块。支持资产的新增、编辑、查询、删除，以及二维码生成、状态变更等功能。

---

## 1. 业务背景

### 1.1 资产卡片定义

资产卡片是固定资产的基本档案记录，每个实物资产对应一张资产卡片，记录其全生命周期信息。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **资产新增** | 录入新购资产信息 |
| **资产编辑** | 修改资产信息（带审批） |
| **资产查询** | 多条件组合查询 |
| **资产详情** | 查看完整资产档案 |
| **二维码** | 生成/打印资产标签 |
| **状态变更** | 入库、在用、报废等状态流转 |

## 3. 数据模型

### 3.1 Asset（资产卡片）核心字段

| 分组 | 字段 | 类型 | 说明 |
|------|------|------|------|
| **基础信息** | asset_code | string | 资产编码（唯一） |
| | asset_name | string | 资产名称 |
| | asset_category | FK | 资产分类 |
| | specification | string | 规格型号 |
| | brand | string | 品牌 |
| **价值信息** | purchase_price | decimal | 购入原值 |
| | purchase_date | date | 购入日期 |
| | supplier | FK | 供应商 |
| **使用信息** | custodian | FK | 保管人 |
| | use_department | FK | 使用部门 |
| | location | string | 存放位置 |
| | asset_status | string | 资产状态 |
| **维护信息** | warranty_date | date | 保修到期 |
| | next_maintenance | date | 下次保养 |

### 3.2 资产状态枚举

| 状态 | 代码 | 说明 |
|------|------|------|
| 待入库 | pending | 已录入未正式入库 |
| 在用 | in_use | 正常使用中 |
| 闲置 | idle | 暂时闲置 |
| 待报废 | pending_disposal | 申请报废中 |
| 已报废 | disposed | 已报废处置 |
| 已调拨 | transferred | 已调拨给其他单位 |

## 4. 核心功能

### 4.1 资产新增

支持多种录入方式：
- **手动录入**：填写完整资产信息
- **复制录入**：基于已有资产复制
- **批量导入**：Excel批量导入
- **扫码入库**：采购扫码自动入库

### 4.2 资产编辑

- 普通字段直接修改
- 关键字段修改需审批
- 记录完整的修改日志

### 4.3 资产查询

支持多维度查询：
- 按分类/部门/状态筛选
- 按购入日期范围
- 按价值范围
- 按关键字搜索

### 4.4 二维码管理

- 自动生成唯一二维码
- 支持批量打印标签
- 二维码包含资产完整信息
- 支持多种标签模板

## 5. 资产编号规则

### 编码格式

```
ZC + YYYYMMDD + 分类码 + 流水号
示例: ZC20240115002001
     │  │        │    │
     │  │        │    └─ 4位流水号
     │  │        └────── 2位分类码
     │  └──────────────── 8位日期
     └──────────────────── 固定前缀(资产)
```

| 分类 | 代码 |
|------|------|

| 通用设备 | 02 |
| 专用设备 | 03 |
| 文物陈列品 | 04 |
| 图书档案 | 05 |
| 家具用具 | 06 |
