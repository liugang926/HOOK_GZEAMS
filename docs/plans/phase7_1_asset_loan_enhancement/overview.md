## 1. 业务背景与痛点

### 1.1 现状分析

GZEAMS在Phase 1.5已实现基础的内部借用功能（AssetLoan模型），支持员工临时借用资产。但在实际业务场景中，存在以下痛点：

| 痛点 | 描述 | 影响 |
|------|------|------|
| **对外借出缺失** | 无法记录借给外部单位/个人的资产 | 资产流向无法追踪，流失风险高 |
| **押金管理缺失** | 外借资产无法记录押金收取和退还 | 财务风险，押金退还无依据 |
| **超期无成本** | 借用超期无任何费用机制 | 资产占用时间长，周转率低 |
| **信用体系缺失** | 无借用人信用记录 | 无法评估风险，重复违约成本低 |

### 1.2 业务场景扩展

本模块在现有内部借用基础上，扩展以下场景：

| 场景 | 说明 | 典型案例 |
|------|------|---------|
| **内部借用（增强）** | 公司员工临时借用，增加超期提醒和计费 | 员工借用笔记本超过30天 |
| **对外借出** | 借给外部单位或个人 | 借给合作单位展示设备、借给客户试用产品 |
| **押金管理** | 外借时收取押金，资产归还后退还 | 借出高价值设备收取押金5000元 |
| **超期计费** | 超期后按天计算费用 | 超期每天收取租金50元 |
| **信用管理** | 记录借用人信用，影响后续借用 | 多次超期归还者降低信用分 |

## 2. 用户角色与权限

### 2.1 角色定义

| 角色 | 权限范围 | 说明 |
|------|---------|------|
| **系统管理员** | 全部权限 | 可配置外部人员对象、计费规则、信用规则 |
| **资产管理员** | 审批借用、管理押金、查看所有借用记录 | 核心操作角色 |
| **仓库管理员** | 确认借出、确认归还、验收资产状况 | 前台执行角色 |
| **财务人员** | 查看押金记录、确认退款、查看计费记录 | 财务对接角色 |
| **普通员工** | 发起内部借用申请、查看自己的借用记录 | 内部使用者 |
| **外部人员** | （无系统账号）通过资产管理员手动创建借用记录 | 外部使用者 |

## 4. 业务流程

### 4.1 内部借用流程（增强）

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│员工发起 │→ │管理员审批│→ │确认借出 │→ │使用中   │→ │确认归还 │
│借用申请 │   │         │   │         │   │(超期提醒)│   │(状况确认)│
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
     │            │            │            │            │
     ▼            ▼            ▼            ▼            ▼
  填写表单     审批通过/拒绝   记录借出时间   超期自动提醒  验收资产
  选择资产     (工作流)       更新资产状态   (通知系统)    更新状态
```

### 4.2 对外借用流程

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│管理员   │→ │创建外部  │→ │填写借用 │→ │收取押金 │→ │确认借出 │
│创建借用 │   │人员记录 │   │单信息   │   │         │   │         │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
     │            │            │            │            │
     ▼            ▼            ▼            ▼            ▼
  选择资产    选择/创建    填写借用    记录押金    记录借出
            ExternalPerson 期限、原因   金额、方式   时间、凭证
                                                │
                                                ▼
                                          ┌─────────┐
                                          │使用中   │
                                          │(超期计费)│
                                          └─────────┘
                                                │
                                                ▼
                                          ┌─────────┐   ┌─────────┐
                                          │确认归还 │→ │退还押金 │
                                          │(状况确认)│  │(扣费结算)│
                                          └─────────┘   └─────────┘
```

### 4.3 超期计费流程

```
定时任务(Celery Beat)每日执行:
│
├── 查询所有超期未归还借用记录
│
├── 判断是否启用计费
│   ├── 内部借用: 按公司政策决定是否计费
│   └── 对外借用: 默认启用计费
│
├── 计算超期天数和费用
│   ├── 获取超期天数 = 当前日期 - 应归还日期
│   ├── 获取计费规则(按日/按阶梯)
│   └── 计算费用
│
├── 创建/更新计费记录
│
└── 发送超期通知(借用人+管理员)
```

### 4.4 信用评分流程

```
借用事件触发信用评分更新:
│
├── 正常归还(未超期): +信用分
├── 超期归还(<=7天): 不变
├── 超期归还(>7天): -信用分
├── 资产损坏: -信用分
├── 资产丢失: -信用分(严重)
│
└── 更新借用人信用记录
    ├── 内部员工: 更新User扩展表
    └── 外部人员: 更新ExternalPerson动态数据
```

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 响应时间 | < 2s | 借用列表查询、详情查看 |
| 并发支持 | 100+ QPS | 仓库多窗口同时操作 |
| 定时任务 | < 5min | 超期检测和计费任务执行时间 |

### 6.2 安全要求

| 要求 | 说明 |
|------|------|
| 数据隔离 | 外部借用记录同样受组织隔离控制 |
| 权限控制 | 外部人员无系统账号，只能通过管理员操作 |
| 审计日志 | 押金收取/退还、信用修改必须有完整审计 |
| 数据脱敏 | 外部人员身份证号等敏感信息脱敏显示 |

### 6.3 兼容性要求

| 要求 | 说明 |
|------|------|
| 向后兼容 | 现有AssetLoan数据不受影响，borrower_type默认为internal |
| 低代码集成 | ExternalPerson对象可通过低代码引擎界面配置 |
| 工作流集成 | 对外借用可配置独立的审批流程 |
