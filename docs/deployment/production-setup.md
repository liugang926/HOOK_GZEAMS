# Production Deployment Guide\n\n## Overview\n\nThis guide provides instructions for deploying the GZEAMS workflow system in a production environment. It covers setting up the necessary infrastructure, configuring services, and deploying the application.\n\n## Architecture\n\n### Components\n\n- **Web Server**: Nginx (reverse proxy, static file serving)\n- **Application Server**: Gunicorn (WSGI HTTP Server for Django)\n- **Database**: PostgreSQL (main data store)\n- **Cache/Broker**: Redis (cache, task queue for Celery)\n- **Asynchronous Tasks**: Celery (background task processing)\n- **Monitoring**: Prometheus, Grafana (metrics collection and visualization)\n- **Logging**: ELK Stack (Elasticsearch, Logstash, Kibana) or similar (centralized logging)\n- **Security**: WAF (Web Application Firewall), IDS/IPS (Intrusion Detection/Prevention System)\n- **Frontend**: NZEAMS Frontend (Vue.js application)\n\n### Deployment Diagram (Conceptual)\n\n```mermaid\ngraph TD\n    User --&gt; |HTTPS| CDN\n    CDN --&gt; |HTTPS| Nginx\n    Nginx --&gt; |HTTP| Gunicorn\n    Gunicorn --&gt; |DB| PostgreSQL\n    Gunicorn --&gt; |Cache| Redis\n    Gunicorn --&gt; |Broker| Redis\n    Gunicorn --&gt; |Logs| Logstash\n    Gunicorn --&gt; |Metrics| Prometheus\n\n    Celery Worker --&gt; |Broker| Redis\n    Celery Worker --&gt; |DB| PostgreSQL\n    Celery Worker --&gt; |Logs| Logstash\n\n    Prometheus --&gt; Grafana\n    Logstash --&gt; Elasticsearch\n    Elasticsearch --&gt; Kibana\n\n    Admin --&gt; Grafana\n    Admin --&gt; Kibana\n\n    subgraph Backend\n        Gunicorn\n        Celery Worker\n    end\n\n    subgraph Data Store\n        PostgreSQL\n        Redis\n    end\n\n    subgraph Monitoring & Logging\n        Prometheus\n        Grafana\n        Logstash\n        Elasticsearch\n        Kibana\n    end\n```\n\n## Prerequisites\n\n- **Operating System**: Ubuntu 22.04 LTS or equivalent Linux distribution\n- **Docker & Docker Compose**: For containerized deployment (recommended)\n- **Python 3.9+**: For Django backend\n- **Node.js 16+**: For Vue.js frontend build\n- **Git**: For source code management\n- **Domain Name**: Configured with DNS records\n- **SSL Certificate**: Let\'s Encrypt or similar\n\n## Setup Steps\n\n### 1. Server Provisioning\n\n- Provision one or more virtual machines or cloud instances.\n- Ensure sufficient CPU, RAM, and disk space based on expected load.\n- Configure firewall rules to allow traffic on ports 80 (HTTP) and 443 (HTTPS).\n\n### 2. Install Docker & Docker Compose\n\nFollow the official Docker installation guide for your OS.\n\n```bash\nsudo apt-get update\nsudo apt-get install ca-certificates curl gnupg\nsudo install -m 0755 -d /etc/apt/keyrings\ncurl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg\nsudo chmod a+r /etc/apt/keyrings/docker.gpg\necho \"deb [arch=\"$(dpkg --print-architecture)\" signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \\\n  \"$(. /etc/os-release && echo \"$VERSION_CODENAME\")\" stable\" | sudo tee /etc/apt/sources.list.d/docker.list &gt; /dev/null\nsudo apt-get update\nsudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin\n\n# Add current user to docker group\nsudo usermod -aG docker $USER\nnewgrp docker\n```\n\n### 3. Clone Repository\n\n```bash\ngit clone https://github.com/your-org/gzeams.git\ncd gzeams\n```\n\n### 4. Environment Configuration\n\nCreate a `.env` file in the project root with sensitive configurations.\n\n```dotenv\n# .env example\n\n# Django settings\nDJANGO_SETTINGS_MODULE=config.settings.production\nSECRET_KEY=your_django_secret_key_here\nDEBUG=False\nALLOWED_HOSTS=your.domain.com,localhost,127.0.0.1\n\n# Database (PostgreSQL)\nPOSTGRES_DB=gzeams_db\nPOSTGRES_USER=gzeams_user\nPOSTGRES_PASSWORD=your_db_password\nPOSTGRES_HOST=db\nPOSTGRES_PORT=5432\n\n# Redis\nREDIS_HOST=redis\nREDIS_PORT=6379\nREDIS_DB=0\n\n# Email (for notifications)\nEMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend\nEMAIL_HOST=smtp.mailtrap.io\nEMAIL_PORT=2525\nEMAIL_USE_TLS=True\nEMAIL_HOST_USER=your_smtp_user\nEMAIL_HOST_PASSWORD=your_smtp_password\nDEFAULT_FROM_EMAIL=noreply@your.domain.com\n\n# Security\nSECURE_SSL_REDIRECT=True\nCSRF_COOKIE_SECURE=True\nSESSION_COOKIE_SECURE=True\n\n# Sentry (for error logging)\nSENTRY_DSN=your_sentry_dsn\n\n# Custom settings for monitoring\nAPM_ENABLED=True\nERROR_TRACKING_ENABLED=True\nAUDIT_LOGGING_ENABLED=True\n\n# Frontend build arguments (if using buildkit to pass secrets)\n# VITE_API_BASE_URL=https://your.domain.com/api\n# VITE_SENTRY_DSN=your_sentry_dsn_frontend\n```\n\n### 5. Build and Deploy with Docker Compose\n\nAdjust `docker-compose.prod.yml` as needed. Example structure:\n\n```yaml\n# docker-compose.prod.yml (simplified example)\nversion: '3.8'\n\nservices:\n  nginx:\n    image: nginx:latest\n    volumes:\n      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro\n      - ./frontend/dist:/var/www/frontend:ro # Serve frontend static files\n      - ./backend/static:/var/www/static:ro   # Serve backend static files\n      - ./backend/media:/var/www/media:ro     # Serve media files\n    ports:\n      - "80:80"\n      - "443:443"\n    depends_on:\n      - web\n    restart: always\n\n  web:\n    build:\n      context: .\n      dockerfile: Dockerfile.backend\n      args:\n        NPM_TOKEN: ${NPM_TOKEN} # If private npm packages are used\n    command: gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 4 --timeout 120\n    expose:\n      - "8000"\n    env_file:\n      - ./.env\n    volumes:\n      - ./backend/static:/app/backend/static\n      - ./backend/media:/app/backend/media\n    depends_on:\n      - db\n      - redis\n    restart: always\n\n  celery_worker:\n    build:\n      context: .\n      dockerfile: Dockerfile.backend\n    command: celery -A config worker -l info --concurrency=4\n    env_file:\n      - ./.env\n    volumes:\n      - ./backend/static:/app/backend/static\n      - ./backend/media:/app/backend/media\n    depends_on:\n      - db\n      - redis\n    restart: always\n\n  db:\n    image: postgres:14-alpine\n    env_file:\n      - ./.env\n    volumes:\n      - postgres_data:/var/lib/postgresql/data/\n    restart: always\n\n  redis:\n    image: redis:6-alpine\n    volumes:\n      - redis_data:/data\n    restart: always\n\nvolumes:\n  postgres_data:\n  redis_data:\n```\n\n**Build and run the services:**\n\n```bash\ndocker compose -f docker-compose.prod.yml build\ndocker compose -f docker-compose.prod.yml up -d\n```\n\n### 6. Database Migrations\n\nApply Django database migrations:\n\n```bash\ndocker compose -f docker-compose.prod.yml exec web python backend/manage.py migrate\n```\n\n### 7. Collect Static Files\n\nCollect Django static files to be served by Nginx:\n\n```bash\ndocker compose -f docker-compose.prod.yml exec web python backend/manage.py collectstatic --noinput\n```\n\n### 8. Frontend Build\n\nBuild the frontend application:\n\n```bash\ncd frontend\nnpm install\nnpm run build\ncd ..\n```\n\n### 9. Configure Nginx for SSL\n\nConfigure Nginx to handle HTTPS and reverse proxy to Gunicorn. Replace `your.domain.com` with your actual domain.\n\n```nginx\n# /nginx/nginx.conf\n\nworker_processes auto;\npid /run/nginx.pid;\ninclude /etc/nginx/modules-enabled/*.conf;\n\nevents {\n    worker_connections 1024;\n}\n\nhttp {\n    sendfile on;\n    tcp_nopush on;\n    tcp_nodelay on;\n    keepalive_timeout 65;\n    types_hash_max_size 2048;\n    include /etc/nginx/mime.types;\n    default_type application/octet-stream;\n\n    # Logging\n    access_log /var/log/nginx/access.log;\n    error_log /var/log/nginx/error.log;\n\n    # Gzip Compression\n    gzip on;\n    gzip_vary on;\n    gzip_proxied any;\n    gzip_comp_level 5;\n    gzip_buffers 16 8k;\n    gzip_http_version 1.1;\n    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;\n\n    # SSL Configuration (assuming certbot or manual setup)\n    # Redirect HTTP to HTTPS\n    server {\n        listen 80;\n        server_name your.domain.com;\n        return 301 https://$host$request_uri;\n    }\n\n    server {\n        listen 443 ssl http2;\n        server_name your.domain.com;\n\n        ssl_certificate /etc/letsencrypt/live/your.domain.com/fullchain.pem;\n        ssl_certificate_key /etc/letsencrypt/live/your.domain.com/privkey.pem;\n\n        ssl_session_cache shared:SSL:10m;\n        ssl_session_timeout 10m;\n        ssl_protocols TLSv1.2 TLSv1.3;\n        ssl_prefer_server_ciphers on;\n        ssl_ciphers \"ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384\";\n\n        # Add HSTS to prevent downgrade attacks\n        add_header Strict-Transport-Security \"max-age=63072000; includeSubDomains; preload\";\n        add_header X-Content-Type-Options nosniff;\n        add_header X-XSS-Protection \"1; mode=block\";\n        add_header X-Frame-Options DENY;\n\n        # Proxy to Django Gunicorn\n        location /api/ {\n            proxy_pass http://web:8000;\n            proxy_set_header Host $host;\n            proxy_set_header X-Real-IP $remote_addr;\n            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;\n            proxy_set_header X-Forwarded-Proto $scheme;\n            client_max_body_size 50M;\n        }\n\n        # Serve frontend static files (Vite build output)\n        location / {\n            root /var/www/frontend;\n            try_files $uri $uri/ /index.html;\n            expires 1d;\n            add_header Cache-Control \"public, max-age=86400\";\n        }\n\n        # Serve backend static files (Django collectstatic)\n        location /static/ {\n            alias /var/www/static/;
            expires 30d;
            add_header Cache-Control \"public, max-age=2592000\";\n        }\n\n        # Serve media files (user uploads)\n        location /media/ {\n            alias /var/www/media/;
            expires 30d;
            add_header Cache-Control \"public, max-age=2592000\";\n        }\n    }\n}\n```\n\n### 10. Configure Celery Beat (Optional - for scheduled tasks)\n\nIf you have scheduled tasks (e.g., periodic reports, cleanups), configure Celery Beat.\n\n```yaml\n# Add to docker-compose.prod.yml\n  celery_beat:\n    build:\n      context: .\n      dockerfile: Dockerfile.backend\n    command: celery -A config beat -l info --scheduler django_celery_beat.schedulers:DatabaseScheduler\n    env_file:\n      - ./.env\n    depends_on:\n      - db\n      - redis\n    restart: always\n```\n\nThen run migrations for `django_celery_beat`:\n\n```bash\ndocker compose -f docker-compose.prod.yml exec web python backend/manage.py migrate django_celery_beat\n```\n\n### 11. Monitoring Integration\n\n- **Prometheus**: Configure Prometheus to scrape metrics from Gunicorn and Celery. Django-Prometheus can be used for this.\n- **Grafana**: Set up Grafana dashboards to visualize metrics from Prometheus and logs from Elasticsearch.\n- **Sentry**: Integrate Sentry for real-time error tracking by setting `SENTRY_DSN` in your `.env`.\n- **Audit Logs**: Ensure audit logs are directed to your centralized logging solution (e.g., ELK Stack) for analysis and alerts.\n\n### 12. Backup Strategy\n\n- Implement regular database backups (PostgreSQL `pg_dump`).\n- Back up media files and application code.\n- Store backups securely off-site.\n\n### 13. Security Best Practices\n\n- **Keep software updated**: Regularly update all components (OS, Docker, Python, Node.js, Django, Vue.js, etc.).\n- **Principle of least privilege**: Grant only necessary permissions to users and services.\n- **Secrets management**: Use environment variables or a secrets manager (e.g., HashiCorp Vault, AWS Secrets Manager) for sensitive data.\n- **Regular security audits**: Perform penetration testing and vulnerability scans.\n- **WAF/IDS/IPS**: Deploy a Web Application Firewall and Intrusion Detection/Prevention Systems.\n\n---\n\n## See Also\n\n- [Monitoring API Documentation](/docs/api/monitoring-api.md)\n- [SLA Dashboard API Documentation](/docs/api/sla-dashboard-api.md)\n- [Audit Logging Documentation](/docs/security/audit-logging.md)\n- [Security Hardening Guide](/docs/security/hardening.md)\n- [Performance Baselines](/docs/reports/performance-baseline.md)
