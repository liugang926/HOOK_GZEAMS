## 一、验收标准概述

### 1.1 验收标准定义

验收标准 (Acceptance Criteria) 是定义功能何时完成的一组明确条件。每个PRD必须包含可验证的验收标准，以便：

- 明确功能完成定义 (Definition of Done)
- 提供清晰的测试依据
- 支持利益相关方验证
- 防止范围蔓延

### 1.2 验收标准格式

使用 **Given-When-Then** 格式定义验收条件：

```gherkin
Given [前置条件]
When [触发操作]
Then [预期结果]
```

## 二、验收标准章节模板

每个PRD应包含以下验收标准章节：

### 2.1 功能验收标准

#### 示例：资产卡片CRUD

| 用户故事 | 验收标准 |
|---------|---------|
| 作为资产管理员，我希望创建新资产卡片，以便记录资产信息 | Given 我已登录系统且有资产创建权限<br>When 我填写完整的资产信息并提交<br>Then 资产卡片创建成功，返回资产ID<br>And 系统自动生成资产编码（格式：ZCYYYYMMNNNN）<br>And 系统自动生成唯一二维码 |
| 作为资产管理员，我希望查看资产列表，以便了解所有资产 | Given 我已登录系统<br>When 我访问资产列表页面<br>Then 显示我有权查看的所有资产<br>And 支持按分类、部门、状态筛选<br>And 支持分页显示（默认每页20条）<br>And 响应时间小于2秒 |
| 作为资产管理员，我希望编辑资产信息，以便更新变更 | Given 我已登录系统且有资产编辑权限<br>When 我修改资产信息并保存<br>Then 资产信息更新成功<br>And 系统记录变更日志（操作人、时间、变更字段） |
| 作为系统，我希望删除资产时使用软删除，以便保留数据 | Given 我已登录系统且有资产删除权限<br>When 我删除一个资产<br>Then 资产被标记为已删除（非物理删除）<br>And 资产在默认列表中不再显示<br>And 管理员可在"已删除"列表中查看和恢复 |

### 2.2 非功能验收标准

| 类别 | 验收标准 | 测量方法 |
|------|---------|----------|
| **性能** | 列表查询响应时间 < 2秒 (95th percentile) | JMeter性能测试 |
| **性能** | 单条记录CRUD响应时间 < 500ms | API性能监控 |
| **性能** | 支持并发100用户同时操作 | 负载测试 |
| **安全** | 所有API端点需要认证 | 安全扫描 |
| **安全** | 跨组织数据完全隔离 | 渗透测试 |
| **可用性** | 系统可用性 ≥ 99.5% | 运维监控 |
| **兼容性** | 支持Chrome最新版、Edge最新版 | 浏览器兼容测试 |

### 2.3 数据验收标准

| 数据项 | 验收标准 |
|--------|---------|
| **组织隔离** | 不同组织的数据完全隔离，跨组织访问返回403 |
| **软删除** | 删除操作为软删除，物理删除需要管理员权限 |
| **审计日志** | 所有数据变更记录创建人、创建时间、更新人、更新时间 |
| **数据完整性** | 关联数据删除时检查引用完整性 |
| **并发控制** | 并发更新同一条记录时，后提交的请求收到冲突提示 |

### 2.4 API验收标准

| API端点 | 验收标准 |
|---------|---------|
| **GET /api/resources/** | 返回标准包装格式 {success, data: {count, results}} |
| **GET /api/resources/{id}/** | 记录不存在时返回404 NOT_FOUND |
| **POST /api/resources/** | 验证失败时返回400 VALIDATION_ERROR |
| **DELETE /api/resources/{id}/** | 使用软删除，返回success: true |
| **批量操作** | 返回summary对象 {total, succeeded, failed} |

## 三、Definition of Done (完成定义)

每个PRD功能模块必须满足以下条件才能认为"完成"：

### 3.1 后端开发DoD

- [ ] 所有模型继承 `BaseModel`
- [ ] 所有序列化器继承 `BaseModelSerializer`
- [ ] 所有ViewSet继承 `BaseModelViewSetWithBatch`
- [ ] 所有过滤器继承 `BaseModelFilter`
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] 所有API响应符合统一格式规范
- [ ] 错误码使用标准错误码
- [ ] 组织隔离测试通过
- [ ] 软删除功能测试通过
- [ ] 批量操作测试通过
- [ ] 代码审查通过
- [ ] 技术文档更新完成

### 3.2 前端开发DoD

- [ ] 使用公共页面组件 (BaseListPage/BaseFormPage)
- [ ] API调用封装在 @/api/ 目录
- [ ] 表单验证完整实现
- [ ] 错误处理使用全局错误组件
- [ ] 组件测试覆盖率 ≥ 70%
- [ ] 跨浏览器兼容性测试通过
- [ ] 响应式布局适配
- [ ] UI/UX设计评审通过
- [ ] 无障碍访问检查通过

### 3.3 集成测试DoD

- [ ] 端到端场景测试通过
- [ ] 前后端联调测试通过
- [ ] 性能测试达标
- [ ] 安全扫描无高危漏洞
- [ ] 用户验收测试 (UAT) 通过

## 四、验收标准示例

### 示例1: 资产分类管理

```markdown
## 验收标准

### 功能验收

| ID | 场景 | Given-When-Then |
|----|------|-----------------|
| AC-001 | 创建资产分类 | Given 我以管理员身份登录<br>When 我创建新的资产分类<br>Then 分类创建成功<br>And 编码自动生成<br>And 可以在分类树中看到新分类 |
| AC-002 | 树形结构显示 | Given 系统中存在多级资产分类<br>When 我查看资产分类页面<br>Then 分类以树形结构展示<br>And 支持展开/折叠<br>And 显示每个分类的资产数量 |
| AC-003 | 删除有子节点的分类 | Given 我尝试删除有子节点的分类<br>When 我点击删除按钮<br>Then 系统提示"该分类下有子分类，无法删除"<br>And 删除操作不执行 |

### 性能验收

- 分类树加载时间 < 1秒（1000个节点）
- 分类拖拽操作响应时间 < 300ms
- 分类搜索响应时间 < 500ms

### 数据验收

- 删除分类时检查是否有资产引用
- 支持最多5级分类层级
- 分类编码全局唯一
```

### 示例2: 用户认证

```markdown
## 验收标准

### 功能验收

| ID | 场景 | Given-When-Then |
|----|------|-----------------|
| AC-101 | 用户登录 | Given 我是注册用户<br>When 我输入正确的用户名和密码<br>Then 登录成功<br>And 获得有效的JWT token<br>And token有效期24小时 |
| AC-102 | 错误密码登录 | Given 我是注册用户<br>When 我输入错误的密码<br>Then 登录失败<br>And 返回401 UNAUTHORIZED<br>And 连续失败5次后账户锁定15分钟 |
| AC-103 | Token过期 | Given 我使用过期的token<br>When 我访问受保护的API<br>Then 请求被拒绝<br>And 返回401 UNAUTHORIZED<br>And 提示token已过期 |

### 安全验收

- 密码使用bcrypt加密存储
- JWT token使用RS256签名
- 登录失败记录审计日志
- 支持强制登出所有设备
```

## 五、验收标准检查清单

在PRD编写完成后，使用此清单验证验收标准的完整性：

### 5.1 完整性检查

- [ ] 是否为每个主要功能编写了用户故事？
- [ ] 验收标准是否使用Given-When-Then格式？
- [ ] 是否包含成功和失败场景？
- [ ] 是否定义了边界条件测试？
- [ ] 是否包含非功能需求验收标准？

### 5.2 可测试性检查

- [ ] 每个验收标准是否可独立验证？
- [ ] 验收标准是否有明确的通过/失败判断？
- [ ] 是否定义了测试数据要求？
- [ ] 是否定义了测试环境要求？

### 5.3 可追溯性检查

- [ ] 验收标准是否可追溯到业务需求？
- [ ] 每个验收标准是否有唯一标识？
- [ ] 验收标准是否与测试用例对应？
- [ ] 是否记录了验收标准的变更历史？
